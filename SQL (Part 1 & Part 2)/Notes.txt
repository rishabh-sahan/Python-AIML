What is a Database
	It is a collection of data in a format that can be easily accessed.

Why Databases?
	> can store large data
	> features like security, scalability etc
	> easier to insert, update, search or delete data

DBMS -> Database management system

SQL v/s NoSQL
	
SQL -> Structured Query Language
	Relational Database (Data stored in Tables)
	Ex: MySQL, Oracle, PostgreSQL etc

NoSQL -> Not Only Structured Query Language
	Non Relational Database (Data stored in document/key-val/graphs etc)
	Ex: MongoDB, Cassandra, Neo4j etc

SQL (Structured Query Language)
	SQL is a programming language used to interact with relational databases.

SQL is case-insensitive

	);

	CREATE TABLE user (
		id INT,
		age INT,
		name VARCHAR(30) NOT NULL,
		email VARCHAR(50) UNIQUE,
		followers INT DEFAULT 0,
		following INT,
		CONSTRAINT CHECK (age >= 13),
		PRIMARY KEY (id)
	);
	
	Constraints -> Rules for data in a table
		NOT NULL -> Column cannot have a null value
		UNIQUE -> All values in column are different
		DEFAULT -> Set the default value for a column
		CHECK -> It can limit the values allowed in a column
		PRIMARY KEY -> Makes a column unique & not null but used only for one
		FORIEGN KEY -> Prevent actions that would destroy links between tables
		
		KEY -> Special Columns
		ER Diagram -> Entity Relationship Diagram

		What are Keys?
			Keys are special columns in the table
		Primary Key
			It is a column (or set of columns) in a table that uniquely identifies each row. (a unique id)
			There is only 1 PK & it should be NOT null & unique
		Foreign Key
			A foreign key is a column (or set of columns) in a table that refers to the primary key in another table. 
			FKs can have duplicate & null values
			There can be multiple FKs

	CREATE TABLE post (
		id INT PRIMARY KEY,

		user_id INT,
		FOREIGN KEY (user_id) REFERENCES user(id)
	);
		
		Where Clause -> To define some conditions 
		SELECT col1, col2 FROM table_name 
		WHERE conditions;

	SELECT * FROM user
	WHERE followers >= 200;

		Where Clause Operators
		Arithmetic Operators -> +, -, *, /, %
		Comparison Operators -> =, !=, >, <, >=, <=
		Logical Operators -> AND, OR, NOT, IN, BETWEEN, LIKE, ALL, ANY
		Bitwise Operators -> &(AND), |(OR)
		
		Where Clause with Frequently Used Operators
		AND -> To check for both conditions to be true
			SELECT name, age, followers FROM user WHERE age >= 15 AND followers >= 200;

		OR -> To check for one of the conditions to be true
			SELECT name, age, followers FROM user WHERE age >= 15 OR followers >= 200;
		
		BETWEEN -> Selects for a given range
			
		IN -> Matches any value in the list
		NOT -> To negate the given condition
