What is a Database
	It is a collection of data in a format that can be easily accessed.

Why Databases?
	> can store large data
	> features like security, scalability etc
	> easier to insert, update, search or delete data

DBMS -> Database management system

SQL v/s NoSQL
	
SQL -> Structured Query Language
	Relational Database (Data stored in Tables)
	Ex: MySQL, Oracle, PostgreSQL etc

NoSQL -> Not Only Structured Query Language
	Non Relational Database (Data stored in document/key-val/graphs etc)
	Ex: MongoDB, Cassandra, Neo4j etc

SQL (Structured Query Language)
	SQL is a programming language used to interact with relational databases.

SQL is case-insensitive

	CREATE TABLE user (
		id INT,
		age INT,
		name VARCHAR(30) NOT NULL,
		email VARCHAR(50) UNIQUE,
		followers INT DEFAULT 0,
		following INT,
		CONSTRAINT CHECK (age >= 13),
		PRIMARY KEY (id)
	);
	
	Constraints -> Rules for data in a table
		NOT NULL -> Column cannot have a null value
		UNIQUE -> All values in column are different
		DEFAULT -> Set the default value for a column
		CHECK -> It can limit the values allowed in a column
		PRIMARY KEY -> Makes a column unique & not null but used only for one
		FORIEGN KEY -> Prevent actions that would destroy links between tables
		
		KEY -> Special Columns
		ER Diagram -> Entity Relationship Diagram

		What are Keys?
			Keys are special columns in the table
		Primary Key
			It is a column (or set of columns) in a table that uniquely identifies each row. (a unique id)
			There is only 1 PK & it should be NOT null & unique
		Foreign Key
			A foreign key is a column (or set of columns) in a table that refers to the primary key in another table. 
			FKs can have duplicate & null values
			There can be multiple FKs

		Where Clause -> To define some conditions 
		SELECT col1, col2 FROM table_name 
		WHERE conditions;

	SELECT * FROM user
	WHERE followers >= 200;

		Where Clause Operators
		Arithmetic Operators -> +, -, *, /, %
		Comparison Operators -> =, !=, >, <, >=, <=
		Logical Operators -> AND, OR, NOT, IN, BETWEEN, LIKE, ALL, ANY
		Bitwise Operators -> &(AND), |(OR)
		
		Where Clause with Frequently Used Operators
		AND -> To check for both conditions to be true
			SELECT name, age, followers FROM user WHERE age >= 15 AND followers >= 200;

		OR -> To check for one of the conditions to be true
			SELECT name, age, followers FROM user WHERE age >= 15 OR followers >= 200;
		
		BETWEEN -> Selects for a given range
			SELECT name, age, followers FROM user WHERE age BETWEEN 15 AND 17;
		
		IN -> Matches any value in the list
			SELECT name, email, followers FROM user WHERE email IN ("adam@gmail.com", "david@gmail.com");
			SELECT name, age, email FROM user WHERE age IN (14, 16);

		NOT -> To negate the given condition
		    SELECT name, age, email FROM user WHERE age NOT IN (14, 16); 

	Limit Clause -> Sets an upper limit on number of (tuples) rows to be returned
	    SELECT col1, col2 FROM table_name
	    LIMIT number;

		SELECT name, age, email FROM user WHERE age > 14
		LIMIT 2;

	Order By Clause -> To sort in ascending (ASC) or descending (DESC) order
	    SELECT col1, col2 FROM table_name
		ORDER BY col_name(s) ASC/DESC;

		SELECT name, age, followers FROM user
		ORDER BY followers ASC;

	Aggregate Functions -> Aggregate functions perform a calculation on a set of values and return a single value.
		COUNT(), MAX(), MIN(), SUM(), AVG() 

		SELECT max(followers) FROM user;
		SELECT min(age) FROM user;
		SELECT count(age) FROM user WHERE age = 14;
		SELECT avg(age) FROM user;
		SELECT sum(followers) FROM user; 

	Group By Clause -> Group rows that have the same values into summary rows.
				It collects data from multiple records and groups the result by one or more columns.
				Generally we use group by with some aggregation functions

		SELECT col1, col2 FROM table_name
		GROUP BY col_name(s);

		SELECT age, COUNT(id) FROM user GROUP BY age;
		SELECT age, max(followers) FROM user GROUP BY age;

	Having Clause -> Similar to where i.e. applies some conditions on rows.
		        But it is used when we want to apply any condition after grouping.
				WHERE is for the table, HAVING is for a group
				Grouping is necessary for HAVING

		SELECT col1, col2 FROM table_name
		GROUP BY col_name(s)
		HAVING condition;

		SELECT age, max(followers) FROM user
		GROUP BY age
		HAVING max(followers) > 200;

	General Order
	SELECT column(s)
	FROM table_name
	WHERE condition
	GROUP BY column(s)
	HAVING condition
	ORDER BY column(s) ASC/DESC;

		SELECT age, max(followers)
		FROM user
		GROUP BY age
		HAVING max(followers) > 200
		ORDER BY age DESC;
